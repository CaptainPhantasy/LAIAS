# Database Migrations

This service uses [Alembic](https://alembic.sqlalchemy.org/) for database migrations.

## Quick Start

```bash
# Navigate to the service directory
cd services/agent-generator

# Run migrations
alembic upgrade head

# Rollback one migration
alembic downgrade -1

# Create a new migration (after modifying models)
alembic revision --autogenerate -m "description of change"
```

## Environment Variables

Alembic reads `DATABASE_URL` from environment or `.env` file:

```bash
DATABASE_URL=postgresql+asyncpg://laias:password@localhost:5432/laias
```

## Migration Files

Migrations are stored in `alembic/versions/`:

- `001_initial.py` - Initial schema from `infrastructure/init.sql`

## Best Practices

1. **Always review autogenerated migrations** before applying them
2. **Test migrations on a copy of production data** before running in production
3. **Create a backup** before running migrations in production
4. **Use descriptive migration messages** for easier tracking

## Docker Environment

When running in Docker, Alembic migrations should be run as part of container startup or as a separate init container:

```bash
# Inside the container
docker exec -it laias-agent-generator alembic upgrade head
```

Or add to the Dockerfile entrypoint:

```bash
alembic upgrade head && uvicorn app.main:app ...
```
